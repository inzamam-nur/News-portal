Some question about javascript

Diference between var,const,let are:

  • The scope of a var variable is functional scope.It can be updated and re-declared into the scope.It can be declared without initialization.It can be accessed without initialization as its default value is “undefined”
  • The scope of a let variable is block scope.It can be updated but cannot be re-declared into the scope.It can be declared without initialization.It cannot be accessed without initialization otherwise it will give ‘referenceError’.
  • The scope of a const variable is block scope.It cannot be declared without initialization. It cannot be accessed without initialization, as it cannot be declared without initialization.

Regular functions created using function declarations or expressions are constructible and callable. Since regular functions are constructible, they can be called using the new keyword. However, the arrow functions are only callable and not constructible, i.e arrow functions can never be used as constructor functions. Function expressions are best for object methods. Arrow functions are best for callbacks or methods like map , reduce , or forEach . Use function declarations for functions you'd call by name (because they're hoisted). Use arrow functions for callbacks (because they tend to be terser).

Diference between foreach() , map(), and Filter() are :

  • .forEach() literally says what it is going to do (similarly .map() and .filter() do pretty much what they say on the tin. But The map() method creates an entirely new array.
  • The forEach() method returns “undefined“. The map() method returns the newly created array according to the provided callback function.
  • foreach() is an ECMAScript5 (ES5) feature. map() and filter() calls as function once for each element in an array

A potent component of contemporary JavaScript introduced in ES6 are template strings. Instead than requiring to concatenate like in earlier JavaScript versions, it enables us to insert/interpolate variables and expressions into strings. It enables us to construct complicated strings with dynamic features. Another great thing that comes with template strings are tags. Tags are functions that take a string and the decomposed parts of the string as parameters and are great for converting strings to different entities.
